# VPC
- AWS上に自分専用のプライベートネットワークを作る仕組みをVPCという
- VPCはリージョン単位で作成する
- VPCを作成するときにCIDRブロック（IPアドレスの範囲）を指定する
- VPC内はサブネットに分割して使う
- サブネットはアベイラビリティゾーン単位で作成する
- インターネットと通信できるサブネットをパブリックサブネットという
- インターネットと通信できないサブネットをプライベートサブネットという
- パブリックサブネットにするにはインターネットゲートウェイをVPCにアタッチする
- インターネットゲートウェイはVPCとインターネットをつなぐ出入り口のこと
- パブリックサブネットにするにはルートテーブルに0.0.0.0/0のルートを追加する
- ルートテーブルはサブネット内のトラフィックをどこに向けるかを定義するもの
- プライベートサブネット内のEC2がアウトバウンド通信するにはNATゲートウェイを使う
- NATゲートウェイはパブリックサブネットに配置する
- NATゲートウェイはアウトバウンド専用で、外からのインバウンド通信は受け付けない
- ステートフルは行きの通信を許可すれば戻りの通信は自動で許可される仕組み
- セキュリティグループはEC2などのリソースレベルのファイアウォールでステートフル
- セキュリティグループはデフォルトで全インバウンドを拒否、全アウトバウンドを許可
- ステートレスは行きと戻りの通信をそれぞれ別に許可ルールを設定する必要がある仕組み
- ネットワークACL（NACL）はサブネットレベルのファイアウォールでステートレス
- ネットワークACL（NACL）はデフォルトですべての通信を許可する
- VPC同士を接続する仕組みをVPCピアリングという
- VPCピアリングは同一リージョン・クロスリージョン・クロスアカウントでも利用できる
- VPCピアリングはA-B-CのときAとCは通信できない
- 複数VPCをつなぐときはTransit Gatewayが適している
- Transit GatewayはVPCやオンプレをハブで一括管理できる
- VPN接続は自社オフィスやデータセンター（オンプレ）とAWSをつなぐ仕組み
- VPNとはインターネット回線を使って安全に通信するトンネルのこと
- 仮想プライベートゲートウェイはAWS側に設置するVPN接続の出入り口
- 仮想プライベートゲートウェイはアタッチして利用する
- カスタマーゲートウェイはオンプレ側の実際のルーターや機器のこと
- カスタマーゲートウェイはAWSにその機器のIPアドレスや設定を登録したもの
- 仮想プライベートゲートウェイとカスタマーゲートウェイの間にVPNトンネルを張る
- VPNトンネルを張ることでオンプレとAWS上のVPCやその内部のEC2などが通信できる
- オンプレとVPCを専用線でつなぐにはDirect Connectを使う
- VPC内のEC2がS3にアクセスするにはインターネット(外部通信)経由のためリスクがある
- VPCエンドポイントを使うとAWS内部の専用ルートでS3などにアクセスできる
- VPC Flow LogsとはVPC内を流れた通信記録(誰/どこ/ポート)をログとして残す仕組み
- VPC Flow Logsの保存先はCloudWatch LogsかS3を選べる
- デフォルトVPCとはAWSアカウントを作ると各リージョンに自動でVPCが1つ用意されている
- 何も設定しなくてもすぐEC2を起動できるのはデフォルトVPCのおかげ
- デフォルトVPCのCIDRは172.31.0.0/16と固定されている
- EC2はVPCの中に置くサーバーだが、S3やDynamoDBはVPC外にあるAWS管理の共有サービス
- 機密データをS3やDynamoDBに保存するとき通信がインターネットに出てしまうとリスク
- S3とDynamoDBはAWSの中でも特に使われる頻度が高いため専用のゲートウェイが存在する
- S3とDynamoDBはゲートウェイ型の通信が推奨されている
- VPCにゲートウェイ型エンドポイントを作成すると、ルートテーブルに自動で経路が追加
- ルートテーブルとはサブネット内の通信をどこに向けるかを書いた地図のようなもの
- 専用ゲートウェイを使うとインターネットゲートウェイがなくてもS3/DynamoDBに接続可
- 専用ゲートウェイを使うとNATゲートウェイがなくてもS3/DynamoDBに接続可
- 専用ゲートウェイ型通信はオンプレから使えない
- 専用ゲートウェイ型通信は無料
- 専用ゲートウェイ型通信はセキュリティグループ非対応
- S3とDynamoDB以外はインターフェース型の通信のみ
- VPCの中にENIが作られてそこにプライベートIPが割り当てられる
- ENIに割り当てられたプライベートIP宛に通信することでAWSサービスに届く
- インターフェース型通信は有料
- インターフェース型通信はセキュリティグループ対応
- {CIDRについては追加の必要あり}
- セキュリティグループはルールに「許可」しか設定できず「拒否」は設定できない
- NACLはルール番号の小さい順に評価され最初にマッチしたルールが適用される
- Direct ConnectはVPNより安定・高速だが開通までに数週間〜数ヶ月かかる
- VPC Flow LogsはリアルタイムではなくCloudWatchやS3に数分遅れで届く
- CIDRのプレフィックス長はVPCを作るときにIPアドレスの範囲の広さを/16〜/28の間で指定
- CIDRのプレフィックス長の/16は約65000個のIPアドレスが使える最大サイズ
- CIDRのプレフィックス長の/28は16個のIPアドレスしかない最小サイズ
- VPのCIDRのプレフィックス長が/16ならサブネットは/17〜/28の間で自由に切れる
- サブネットを作るとAWSが自動で5つのIPアドレスを内部利用のために確保する
